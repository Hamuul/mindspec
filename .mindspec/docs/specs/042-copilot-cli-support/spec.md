# Spec 042-copilot-cli-support: Copilot CLI Integration Support

## Goal

Make MindSpec a first-class workflow for GitHub Copilot CLI users so teams can run Spec -> Plan -> Implement -> Review with Copilot, and run brownfield migration classification without requiring Claude CLI.

## Background

MindSpec is CLI-first and already documents Claude Code and Codex workflows, but Copilot CLI is not yet a first-class documented/bootstrap target.

Two practical gaps currently block smooth Copilot adoption:

1. **Workflow onboarding gap**: there is no dedicated Copilot quick-start guide and no Copilot-specific bootstrap artifacts generated by `mindspec init` (for example `.github/copilot-instructions.md` and repository hooks).
2. **Migration provider gap**: `mindspec migrate` LLM classification is effectively Claude-bound today (probe and executor paths), which blocks Copilot-only environments from using assisted classification.

This spec adds first-class Copilot onboarding and migration-provider support while preserving existing Claude/Codex behavior and MindSpec's CLI-centric architecture.

## Impacted Domains

- **workflow**: Copilot operating guide and phase transitions using direct `mindspec` commands
- **core**: bootstrap artifacts and top-level docs/reference updates
- **brownfield migration subsystem**: provider resolution and LLM classification execution path

## ADR Touchpoints

- [ADR-0003](../../adr/ADR-0003.md): `mindspec instruct` remains the canonical runtime instruction source; Copilot bootstrap must defer to CLI-emitted guidance.
- [ADR-0012](../../adr/ADR-0012.md): Copilot integration must compose with external CLIs and repository-native config files instead of wrapping or replacing Copilot.

## Requirements

### 1. Copilot workflow onboarding

1. Add a dedicated Copilot guide at `.mindspec/docs/user/guides/copilot-cli.md`.
2. The guide must include: setup prerequisites, session-start guidance pattern, direct workflow commands (`mindspec spec-init`, `mindspec approve ...`, `mindspec next`, `mindspec complete`), and known differences from Claude/Codex integrations.
3. Update user-facing entry points (README and guide index references) so Copilot is listed alongside Claude and Codex.
4. Extend `mindspec init` scaffolding to add `.github/copilot-instructions.md` with minimal bootstrap guidance that points to `mindspec instruct`.
5. Extend `mindspec init` scaffolding to add a repository hook template under `.github/hooks/` that runs `mindspec instruct` on session start (or equivalent supported hook event).
6. Bootstrap creation must remain additive and non-destructive: existing Copilot files are never overwritten.

### 2. Brownfield migrate provider support

7. Add explicit `copilot-cli` provider support to migration classification.
8. Classification execution must be provider-routed (not hardwired to `claude`), so the selected provider determines which CLI is executed.
9. When `MINDSPEC_LLM_PROVIDER=copilot-cli`, migrate plan/apply must not execute a Claude probe or Claude classifier call.
10. Provider auto-detection when `MINDSPEC_LLM_PROVIDER` is unset must support Copilot CLI availability (with deterministic precedence documented in code and tests).
11. Unsupported/missing provider errors must be explicit and actionable, including remediation via `MINDSPEC_LLM_PROVIDER` and `MINDSPEC_LLM_MODEL`.

### 3. Quality and compatibility

12. Existing Claude-based migrate behavior remains backward compatible when Claude is selected.
13. Existing deterministic-only mode (`MINDSPEC_LLM_PROVIDER=off`) remains unchanged.
14. New and updated tests cover provider resolution, probe behavior, and classification command routing.
15. `make test` passes.

## Scope

### In Scope
- Copilot documentation and quick-start path in user docs/README
- Bootstrap changes in `mindspec init` for Copilot instruction/hook artifacts
- Migration provider resolution and classifier execution path updates
- Unit/integration tests for migrate provider behavior

### Out of Scope
- Full AgentMind telemetry setup automation specifically for Copilot
- Redesign of MindSpec workflow modes, state model, or Beads lifecycle
- GitHub-hosted coding-agent policy configuration outside repository-controlled files

## Non-Goals

- Replacing or regressing existing Claude/Codex integrations
- Introducing Copilot-specific slash commands instead of direct `mindspec` CLI usage
- Building a generic multi-provider SDK beyond what migrate classification currently needs

## Acceptance Criteria

- [ ] A Copilot guide exists at `.mindspec/docs/user/guides/copilot-cli.md` and is linked from user entry-point docs.
- [ ] `mindspec init` creates Copilot bootstrap artifacts (`.github/copilot-instructions.md` and hook template) additively.
- [ ] Running `mindspec migrate plan` with `MINDSPEC_LLM_PROVIDER=copilot-cli` routes LLM classification through Copilot CLI and does not invoke Claude.
- [ ] Running `mindspec migrate plan` with `MINDSPEC_LLM_PROVIDER=off` performs deterministic-only classification with no LLM CLI execution.
- [ ] Running migrate with unset provider on a Copilot-only environment can resolve/use Copilot provider per documented precedence.
- [ ] Existing Claude migrate tests remain green.
- [ ] New Copilot provider tests are present and passing.
- [ ] `make test` passes.

## Validation Proofs

- `go test ./internal/brownfield -run 'ResolveLLMConfig|Copilot|Classify'`: provider detection/routing tests pass for Copilot, Claude, and off modes.
- `go test ./internal/bootstrap -run 'Init|Copilot'`: init scaffolding tests confirm additive Copilot file creation.
- `MINDSPEC_LLM_PROVIDER=off ./bin/mindspec migrate plan --run-id 042-off-smoke`: plan completes without invoking an LLM provider.
- `MINDSPEC_LLM_PROVIDER=copilot-cli ./bin/mindspec migrate plan --run-id 042-copilot-smoke`: low-confidence docs are classified via Copilot path (when Copilot CLI is available).
- `make test`: full suite passes.

## Open Questions

*None. Resolved decisions:*
- Copilot support is CLI-first and reuses `mindspec instruct` as canonical runtime guidance.
- Brownfield migrate gets explicit provider-level Copilot support instead of a Claude-only implicit path.
- Copilot bootstrap is repository-native (`.github/copilot-instructions.md` + `.github/hooks/`) and additive.
